# DBMS Practical

A comprehensive collection of SQL practical exercises covering various Database Management System concepts and operations.

## 📚 Contents

This repository contains SQL files demonstrating different database concepts:

1. **Creating tables ARD.sql** - Database and table creation with sample data
2. **Data constraints ARD.sql** - Implementation of various SQL constraints
3. **Aggregates Function and Group By ARD.sql** - Aggregate functions and GROUP BY operations
4. **String and Set Operation, Order By ARD.sql** - String manipulation, set operations, and sorting
5. **Type of Joints ARD.sql** - Various types of SQL joins (INNER, LEFT, RIGHT, FULL)
6. **Subqueries in Form and With Clause ARD.sql** - Nested queries and WITH clause
7. **Null Values and Null Values (Cont.) ARD.sql** - Handling NULL values in SQL
8. **Exists and Not Exists ARD.sql** - EXISTS and NOT EXISTS clause implementations
9. **SOME and ALL clause ARD.sql** - SOME and ALL operators
10. **Views in SQL ARD.sql** - Creating and managing SQL views
11. **SQL Triggers ARD.sql** - Database triggers and automation

## 🚀 Getting Started

### Prerequisites
- MySQL Server installed on your system
- MySQL Workbench or any SQL client

### Usage

1. Clone this repository:
```bash
git clone https://github.com/Arjunyaa/DBMS-Practical.git
```

2. Open your SQL client (MySQL Workbench, phpMyAdmin, etc.)

3. Execute the SQL files in sequence or individually based on the concept you want to practice

4. Each file contains:
   - Database/Table creation statements
   - Sample data insertion
   - Query examples demonstrating the concept

## 📝 Topics Covered

- **DDL (Data Definition Language)**: CREATE, ALTER, DROP
- **DML (Data Manipulation Language)**: INSERT, UPDATE, DELETE
- **DQL (Data Query Language)**: SELECT with various clauses
- **Constraints**: PRIMARY KEY, FOREIGN KEY, UNIQUE, NOT NULL, CHECK
- **Joins**: INNER, LEFT, RIGHT, FULL OUTER
- **Subqueries**: Nested queries, correlated subqueries
- **Aggregate Functions**: COUNT, SUM, AVG, MIN, MAX
- **Set Operations**: UNION, INTERSECT, EXCEPT
- **Advanced Concepts**: Views, Triggers, EXISTS, ALL, SOME

## 💡 Sample Database Structure

The practicals use a `college_db` database with tables including:
- **students**: Student information (id, name, age, course, city, marks)
- **instructor**: Faculty details (id, name, department, salary)
- **alumni**: Alumni records (PRN, Name)
